# 如何使用.gitignore文件删除掉已经提交的文件
## 问题描述
最近在整理一个老项目的时候发现项目中有一些无用的文件被提交到代码管理仓库了，于是我加了.gitignore文件，但是已经提交的文件就不会受到.gitignore文件约束了，解决办法是把已经提交的删除掉。

## 具体操作步骤
- 首先如果本次有没提交的修改先提交。
- 依次执行下面3个命令:
```
git rm -r --cache .
git add .
git commit -m "rm laji"
```
经过上面几步后垃圾文件就被清理了。但是本地文件不会被删除,但是也不会提交到版本库中。
